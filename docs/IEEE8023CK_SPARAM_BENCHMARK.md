# IEEE 802.3ck S-Parameter Benchmark

This project now includes a COM-style, IEEE 802.3ck-inspired benchmark path
for public Touchstone channels.

It is not an IEEE compliance test. It does not run COM, ERL, VEC, or any
formal TP pass/fail procedure. It is a simulation benchmark that uses:

```text
PAM4 symbols
-> Tx FFE proxy
-> Touchstone S-parameter channel converted to a symbol-rate impulse response
-> AWGN and optional crosstalk-like stress
-> receiver equalizers
-> BER / SER / eye metrics / Markov routing metrics
```

## Where to Put Channels

Download public IEEE 802.3ck contributed channel files and place `.s4p` or
other `.sNp` Touchstone files under:

```text
data/8023ck_channels/
```

Recommended subfolders:

```text
data/8023ck_channels/C2M/
data/8023ck_channels/C2C/
data/8023ck_channels/Backplane/
data/8023ck_channels/Cable/
```

The runner can auto-discover files recursively. For paper-grade runs, prefer
an explicit manifest:

```text
data/8023ck_channels/channel_manifest.csv
```

Columns:

```text
case_id,group,role,file,baud,loss_db,include
C2M_10dB,C2M,main,C2M/c2m_10db.s4p,26.5625e9,10,1
C2M_14dB,C2M,main,C2M/c2m_14db.s4p,26.5625e9,14,1
C2M_16dB,C2M,main,C2M/c2m_16db.s4p,26.5625e9,16,1
C2C_20dB,C2C,main,C2C/c2c_20db.s4p,26.5625e9,20,1
BP_24dB,Backplane,stress,Backplane/bp_24db.s4p,26.5625e9,24,1
Cable_2m_XTALK,Cable,stress,Cable/cable_2m.s4p,26.5625e9,28,1
```

## How to Run

Smoke test, using synthetic fallback channels if no Touchstone files exist:

```matlab
out = run_paper('8023ck_sparam', 'trials', 1, 'snr', 22);
```

Paper run after adding real IEEE-contributed Touchstone files:

```matlab
out = run_paper('8023ck_sparam', ...
    'channel_dir', 'data/8023ck_channels', ...
    'trials', 10, ...
    'snr', [18 22 26 30], ...
    'max_cases', 12, ...
    'run_markov', true, ...
    'save_dir', 'paper_figs');
```

The summary CSV is written to:

```text
paper_figs/ieee8023ck_sparam_benchmark_summary.csv
```

## What It Evaluates

The static benchmark compares:

- No EQ
- NLMS-DFE
- SMNLMS-DFE
- SM-sign-NLMS
- Liu-style SS-LMS DFE
- Chen-style pulse-response FFE/DFE reference
- ExtraTrees-HMM-inspired classifier/router
- Algorithm 1
- Proposed MSB

Metrics:

- BER
- SER
- eye height
- eye width

The Markov benchmark selects three channels from the loaded set and switches
among them with slow, medium, and fast transition matrices. It reports:

- Algorithm 6 BER
- Algorithm 1 BER
- MSB state accuracy

## Paper Wording

Use conservative language:

> The channel benchmark is constructed from publicly available IEEE 802.3ck
> contributed S-parameter channels. The evaluation follows a COM-style PAM4
> simulation flow but does not claim full IEEE compliance testing.

For Markov switching:

> The Markov disturbance is generated by switching among S-parameter-derived
> channel impulse responses instead of using synthetic scalar ISI taps.
