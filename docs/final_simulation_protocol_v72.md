# Final Simulation Protocol v72

This protocol separates the theory-first validation from the realistic C2M
tracking application.  The wording is intentionally conservative: the 802.3ck
cases are COM-style simulations using public S-parameter channels, not IEEE
compliance tests.

## Simulation Blocks

### Block A: Controlled Severe Markov-ISI Validation

Purpose: show the proposed tracking architecture under the synthetic severe
Markov channel where the theory assumptions are clearest and the gain is
largest.

Channel model:
- PAM4
- Markov two-tap ISI
- h2 states are generated by `controlled_markov_isi_profile_v72`
- severe profile: h2 = {0.30, 0.50, 0.70}
- transition matrix is generated from mean dwell length 20 symbols:
  p_stay = 1 - 1/20 = 0.95, giving
  [0.95 0.05 0; 0.025 0.95 0.025; 0 0.05 0.95]

Parameter provenance:
- The center state h2 = 0.50 is used as a controlled two-tap
  partial-response target, consistent with the common 1+0.5D PAM4
  partial-response / TX-DFE stress model in high-speed SerDes literature.
  A representative hardware example is IBM/VLSI Circuits 2018,
  "A 0.3pJ/bit 112Gb/s PAM4 1+0.5D TX-DFE Precoder and 8-Tap FFE in
  14nm CMOS".
- The outer states h2 = 0.30 and h2 = 0.70 are a symmetric low/high
  postcursor-burden span around that target.  They are not claimed as
  measured 802.3ck channels.
- Block B is the measured/contributed C2M S-parameter application benchmark.

Methods:
- Algorithm 2 / Proposed HMM-MSB
- Oracle MSB (printed only)
- Algorithm 1
- NLMS
- SMNLMS
- SM-sign-NLMS
- Liu SS-LMS
- Chen pulse-ref
- Cui HMM adapted

Metrics:
- BER / SER vs SNR, 15 to 30 dB
- Monte Carlo mean BER curves
- optional ratio table vs key baselines

Interpretation:
- This is the theory-aligned stress test.
- It is not a realistic 802.3ck channel benchmark.
- Use this block to show why multi-state memory and state routing matter.

### Block B: C2M S-Parameter Tracking Stress Benchmark

Purpose: main application benchmark for the proposed MSB receiver.

Channel states:
- C2M_10dB
- C2M_14dB
- C2M_16dB

Stress profiles:
- slow
- medium
- fast

Recommended wording:

> The channel is piecewise constant and switches among low-, medium-, and
> high-loss C2M S-parameter responses. This tracking stress test is not part
> of the IEEE COM compliance procedure; it evaluates online receiver tracking
> under recurring channel-state variations.

Methods:
- Proposed MSB
- Algorithm 1
- Chen pulse-ref

Metrics:
- BER / SER
- transition-window BER
- recovery time
- state accuracy / wrong-routing rate
- bank usage
- eye height
- eye width

Interpretation:
- This block links the theory-driven Markov tracking idea to a realistic
  C2M S-parameter application.
- The focus is adaptation/tracking stress, not static COM compliance.

### Block C: Endogenous-Aware Recursion Bridge

Purpose: show why the endogenous-aware theory is useful before introducing
the full MSB architecture.

Methods:
- NLMS
- SMNLMS
- SM-sign-NLMS
- Endogenous-aware NLMS (ours)

Benchmark:
- controlled Markov tracking stress
- same `controlled_markov_isi_profile_v72` severe profile as Block A

Metrics:
- BER / SER vs SNR, 15 to 30 dB
- tail MSE
- set-membership update rate
- endogenous burden proxy

Interpretation:
- Endogenous-aware NLMS is the single-bank bridge between SMNLMS/SM-sign
  filtering and the full MSB receiver.
- Proposed MSB is evaluated separately in Block A/B to avoid hiding the
  endogenous-aware ablation.

## Suggested Paper Claims

Use for Block A:

> In the controlled severe Markov-ISI validation, the proposed MSB receiver
> substantially reduces BER because it maintains state-specific equalizer
> memory instead of forcing one adaptive filter to learn a compromise channel.

Use for Block B:

> Under the C2M S-parameter tracking stress benchmark, the proposed MSB
> receiver reuses state-specific equalizer banks, while single-bank adaptive
> receivers must continuously relearn a compromise solution. This explains the
> BER, transition-window BER, recovery-time, and eye-opening improvements.

Use for Block C:

> The endogenous-aware NLMS ablation improves over conventional SMNLMS and
> SM-sign-NLMS by accounting for decision-directed reliability and endogenous
> error burden. This supports the endogenous-bias component before the full
> multi-state-bank architecture is introduced.
